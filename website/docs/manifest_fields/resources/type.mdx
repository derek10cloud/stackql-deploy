Resource type, values include : `resource` (default), `query`, `script`

<File name='stackql_manifest.yml'>

```yaml
resources:
- name: get_subnets
  type: query
...
```

</File>

:::info

- `resource` will typically include `create`, `update`, `delete`, `preflight`, `postdeploy` and `exports` methods and is intended for provisioning or configuring a resource  
- `query` is designed to return data using an `exports` method
- `script` is used to incorporate an external script in your stack definition (non StackQL query)

:::
